= Upgrade Health Checker를 사용하여 업그레이드 계획 생성
:hardbreaks:
:allow-uri-read: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Upgrade Health Checker를 사용하여 단일 또는 여러 ONTAP 클러스터에 대한 업그레이드 계획을 생성할 수 있습니다.

또한 여러 클러스터로 구성된 크고 복잡한 환경을 운영하는 경우 최신 업그레이드 계획을 유지 관리하는 데 도움을 받기 위해 link:automate-report-upgrade-health-checker.html["업그레이드 보고서 생성 자동화"]할 수도 있습니다.

.시작하기 전에
특정 Upgrade Health Checker 매개변수 및 도구가 입력을 수락하는 우선순위 순서에 대한 자세한 내용은 link:upgrade-health-checker-input-parameters.html["Upgrade Health Checker의 입력 매개변수에 대해 알아보십시오"]을 참조하십시오.

.단계
. 처음 실행하기 전에 바이너리에 대한 실행 권한을 설정하십시오.
+
[source, cli]
----
chmod +x uhc
----
+

NOTE: Upgrade Health Checker는 독립 실행형 패키지이므로 실행하기 전에 압축을 풀어야 합니다. 이 과정은 몇 초 정도 소요될 수 있습니다.

. Upgrade Health Checker를 완전히 실행하기 전에 이 툴이 클러스터 및 필수 엔드포인트에 연결할 수 있는지 확인하기 위해 종합적인 검사를 수행하십시오.
+
[source, cli]
----
--test all
----
+
종합적인 점검을 통해 Upgrade Health Checker를 호스팅하는 가상 머신이 다음을 갖추고 있는지 확인합니다.

+
** HTTPS를 통한 ONTAP 클러스터 IP 주소 연결
** HTTPS를 통한 원격 측정 엔드포인트((https://support.netapp.com/[]) 연결
** HTTPS를 통한 자동 업데이트 엔드포인트((https://api.uhc.netapp.com[] 및  https://gql.aiq.netapp.com[])에 대한 연결
** 최소 4GB의 여유 `/tmp` 공간


. (선택 사항) 구성 파일을 사용하여 매개변수를 저장하려면 바이너리를 다운로드한 위치와 동일한 위치에 있는  `config.yaml.example` 파일의 이름을  `config.yaml`로 변경하십시오.
+
다음은 config.yaml 파일의 예입니다.

+
[listing]
----
# Application Configuration
APP:
  RUNS_PATH: "/opt/uhc/runs"

# Cluster Credentials
CLUSTER:
  IP: "x.x.x.x"
  USERNAME: "admin"
  PASSWORD: "xyz"
  TARGET_ONTAP_VERSION: ""  # Optional: Specify target ONTAP version (e.g., "9.16.1" or "current" to keep existing version). Leave empty to prompt user.
  ACCEPT_EULA: false  # Optional: Set to true to accept EULA through config. If false/empty, user will be prompted interactively.
----
. 업그레이드할 ONTAP 버전을 입력하고 EULA에 동의한 후 필요에 따라 추가 매개변수를 포함하여 Upgrade Health Checker를 실행하십시오.
+

NOTE: EULA는 여러 페이지로 구성된 문서입니다.  `all`를 입력하여 전체 문서를 한 번에 보고  `y`를 입력하여 동의하십시오.

+
클러스터의 AutoSupport 로그가 다운로드된 후 처리에는 일반적으로 클러스터 노드 수에 따라 1~2분이 소요됩니다. ONTAP 소프트웨어 버전마다 AutoSupport 수집 방식이 다르기 때문에 업그레이드 상태 검사기 실행 시간은 클러스터의 현재 부하와 현재 ONTAP 버전에 따라 달라집니다.

+
.. link:upgrade-health-checker-input-parameters.html["Upgrade Health Checker의 입력 매개변수에 대해 알아보십시오"]을 참조하여 사용 가능한 Upgrade Health Checker 매개변수, 입력 우선순위 순서 및 특정 매개변수의 기본값을 숙지하십시오.
+
입력 매개변수 가이드에서는 워크플로를 간소화하기 위해 사용자 지정 구성 파일 경로와 실행 경로를 지정하는 방법에 대한 정보도 제공합니다. 보고서와 로그를 효율적으로 정리하려면 각 클러스터에 대한 사용자 지정 출력 디렉터리를 만드는 것이 좋습니다.

.. EULA를 수락하고 대상 ONTAP 버전 및 추가 매개 변수를 입력하여 업그레이드 상태 검사기를 실행합니다.
+
[source, cli]
----
./uhc
----


. 업그레이드 상태 검사기가 검사를 완료하면  `runs` 폴더로 이동하여 업그레이드 계획 및 클러스터 보고서를 확인하십시오.
+

NOTE: 프로그램을 실행할 때마다 "고유한 실행"이 되며 관련 `runs` 폴더에 모든 관련 데이터, 로그 및 결과가 포함됩니다.

. 웹 브라우저에서 보고서 파일( `uhc_<cluster-name>_<YYYYMMDDHHMMSS>.html`)을 열어 보고서를 확인하십시오. 원격 호스트에서 실행하는 경우 먼저 보고서 파일을 웹 브라우저에서 보고서를 볼 수 있는 시스템으로 다운로드하십시오.
+
로그 경로 및 보고서 경로는 다음과 같습니다.

+
** 로그 경로: `<output-path>/<unique-run-dir>/<cluster-name>/logs`
** 보고서 경로: `<output-path>/<unique-run-dir>/<cluster-name>/results/uhc_<cluster-name>_<YYYYMMDDHHMMSS>.html`
